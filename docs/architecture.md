# Architecture

## Problem

Define one clear Bharat-relevant AI pain point with concrete users and measurable value.

## Solution overview

Describe the end-to-end flow from input to user-visible output in 6-10 lines.

## Component lanes

- Application layer (`projects/main-submission/src`)
- Model and data integration layer
- Evaluation and reliability layer
- Demo and presentation layer (`docs/`, `assets/`)

## Quality bars

- Runnable on a clean machine with documented setup.
- Reproducible demo path in under 5 minutes.
- Explicit failure modes and fallback behavior.

## Risks

- Over-scoping beyond what can be demoed reliably.
- Weak link between technical work and judging narrative.
- Late integration issues across teammate branches.
