# Submission Checklist

## Must-have

- [ ] README explains problem, solution, setup, and run flow.
- [ ] Active implementation is in `projects/main-submission/`.
- [ ] Demo script is validated and timed.
- [ ] Architecture summary matches implementation reality.
- [ ] Submission package includes required source and run instructions.

## Judging alignment

- [ ] Impact claim is concrete and testable.
- [ ] Innovation is explicit versus baseline alternatives.
- [ ] Technical choices are defensible.
- [ ] UX flow is understandable without author guidance.
- [ ] Presentation quality is polished (README + visuals + demo clarity).

## Final checks

- [ ] All links work.
- [ ] No secrets committed.
- [ ] Repo contains only hackathon-relevant code/docs/assets.
