## Event positioning and expectations

The challenge description frames AI for Bharat as a practical, execution-first hackathon with real problem statements from government and industry. It emphasizes usable solutions over conceptual-only ideas.

## Implications for this repo

- Keep one stable demo candidate in `projects/main-submission/`.
- Use `projects/experiments/` for risky spikes.
- Tie each major feature to at least one judging-relevant outcome (impact, technical depth, usability, presentation clarity).
- Keep run instructions and demo path current at all times.
